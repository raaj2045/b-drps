For each paper, only its metadata is stored on-chain — the author name
(24 characters), e-mail address (29), title (120), abstract (1,500),
reviewer report (2,500), editor report (1,000), and decision letter (300),
together with the 80-character IPFS gateway link whose content identifier
(CID) references the manuscript, which itself is stored off-chain — so each
paper occupies a fixed ~33 KiB of storage (six copies retained across the
pipeline's archival arrays) and total storage grows linearly with the
number of papers, independent of manuscript size. The slot-level accounting
underlying the figure was validated against an SSTORE count obtained via
debug_traceTransaction, with exact agreement (1,167 = 1,167 slots per
publication lifecycle).
